Dashboard notes

How this dashboard is built — and what it could become

A working brief on the data we pull, the sources we deliberately don't pull, and the next moves worth considering.

What's tracked

  • 8 competitors — Eltropy, Kasisto, Posh, Glia, Active.Ai, Omilia, Gridspace, Born Digital.
  • 6 banking regulators — CFPB, Federal Reserve, OCC, FDIC, NCUA, Treasury. Foreign-policy items (sanctions, Iran, tariffs) are filtered out.
  • Industry coverage — TechCrunch AI, Finextra, American Banker, Product Hunt AI, Hacker News, Google News.
  • Refresh cadence — daily at 07:00 PT via GitHub Actions; data commits straight back to the repo, the site picks it up automatically.

Constraints & how to overcome them

Twitter / X

Not currently a source. X's API requires a paid plan starting at ~$100/month even for read-only search. Free scraping is rate-limited and blocks our automation IPs.

How to enable: sponsor a basic API seat ($100/mo) or wait for X to restore free read access. Adding it would re-introduce live conversation from analysts and AI-Twitter.

Bluesky

Attempted, then pulled. The public search endpoint now requires authentication; even with credentials, the signal-to-noise on competitor names was poor — short posts containing words like "posh" or "active AI" leaked into the competitor feed.

How to enable: two paths. (a) curate a list of follow-accounts (analysts, vendors, banking AI thinkers) and only pull their posts — trades breadth for precision. (b) add a semantic-relevance step using an LLM to filter Bluesky posts before they enter the feed (~$1/month at current volume).

Reddit

Reddit blocks our automation's IPs at the API layer. We currently route via a public mirror (pullpush.io), which works but adds a delay and depends on a community-run service.

How to harden: move to Reddit's own OAuth flow — free, but requires registering an app and storing two more secrets. Worth it if pullpush starts to fail intermittently.

Competitor blogs

Most competitor blogs aren't on standardized RSS paths. Today we pull from Omilia, with the rest as TODO — their site search/feed URLs need to be probed one-by-one.

How to fill in: for each competitor, find their actual blog URL (Eltropy, Kasisto, Posh, Glia, Gridspace, Born Digital) and add the verified RSS URL to COMPETITOR_BLOG_FEEDS.

Possible enhancements

  1. AI daily brief at the top. Claude reads everything we pulled and writes "3 things you need to know today" in plain English. ~$1–2/month in API calls. Highest leverage: turns a feed into a briefing.
  2. Email or Slack digest at 07:00. Same content auto-delivered before your standup — you scan before the dashboard ever opens.
  3. Customer-win tracker. Dedicated pattern match for "[Competitor] signs [Bank/CU]" across all sources, surfaced as a separate sub-tab.
  4. Hiring velocity tracker. Scrape competitor /careers pages; new conversational-AI engineer postings are a leading indicator of where they're investing next.
  5. Earnings call mentions. Bank and CU earnings transcripts (via Seeking Alpha / Capital IQ) often name vendor partners. Surface every mention of our competitor set.
  6. Sentiment over time. Track tone of social/news mentions per competitor as a small chart per item set — useful for spotting momentum shifts before they're newsworthy.
